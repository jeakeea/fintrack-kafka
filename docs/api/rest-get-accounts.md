## Описание

| Параметр | Значение |
|----------|----------|
| **URL\*** | `/accounts` |
| **Метод HTTP\*** | `GET` |
| **Наименование метода\*** | Получение списка счетов с балансами |
| **Частичный запрос атрибутов\*** | Нет |
| **Постраничный вывод\*** | Нет |
| **Возврат информации о количестве записей (count)\*** | Нет |
| **Управление сортировками\*** | Нет |
| **Сортировка по умолчанию\*** | Идентификатор счёта (`account_id`), по возрастанию |
| **Способ авторизации\*** | Без авторизации (аутентификация клиента вне scope учебного стенда, см. FR-W1) |
| **Роли\*** | Для всех ролей |
| **Ссылка на swagger** | http://localhost:8000/docs (живой Swagger UI gateway); контракт – [`openapi.yaml`](../openapi.yaml) |
| **http-статус коды в ответе\*** | 200 – ок; Ошибки: 500 |

## Входные параметры\*

Метод не принимает входных параметров.

## Пример запроса

```
GET http://localhost:8000/accounts
```

## Выходные параметры\*

Ответ – массив объектов «счёт с балансом».

| Параметр | Тип | Описание | Обязательный | Пример |
|----------|-----|----------|--------------|--------|
| [ ].account_id | string | Идентификатор счёта | Да | `acc_ivan` |
| [ ].owner_name | string | Имя владельца счёта | Да | `Иван Смирнов` |
| [ ].balance | integer (int64) | Текущий баланс в минорных единицах (копейки); может быть `0` | Да | `850000` |

## Пример ответа

```json
[
  { "account_id": "acc_ivan",  "owner_name": "Иван Смирнов",  "balance": 850000 },
  { "account_id": "acc_maria", "owner_name": "Мария Петрова", "balance": 650000 },
  { "account_id": "acc_oleg",  "owner_name": "Олег Сидоров",  "balance": 500000 }
]
```

## Логика работы

1. Прочитать представление `balances` с сортировкой по `account_id`.
2. Вернуть список счетов с вычисленными балансами.

Баланс не хранится отдельным полем – представление `balances` вычисляет его
из журнала двойной записи: `SUM(credit) − SUM(debit)` по проводкам счёта в
`ledger_entries` (`LEFT JOIN` + `COALESCE(..., 0)` дают `0` для счёта без
операций). Поэтому баланс всегда согласован с историей проводок и не требует
отдельной синхронизации (см. [логическую модель](../data-model/logical-model.md)
и [ADR-0005](../adr/0005-money-as-minor-units.md)).

## Коды ответа\*

| HTTP-статус | Код сообщения | Сообщение | Сценарий возникновения |
|-------------|---------------|-----------|------------------------|
| 200 | OK | Операция выполнена успешно | Список счетов возвращён (возможно, пустой) |
| 500 | InternalServerError | Сервис временно недоступен | Внутренняя ошибка сервера (например, недоступна БД) |
